time-wait为什么需要？为什么是2MSL
|---
第一个MSL保证了主动关闭方发出的最后一个ACK有足够时间到达对方，若该包丢失，
第二个MSL则预留了给对方重传FIN并安全抵达的时间，从而避免连接关闭异常；
同时，这2MSL的时长能让网络中所有因路径延迟产生的过期报文自然结束，确保后续建立的相同端口连接不会被这些旧数据所干扰。

time-wait高并发场景会带来什么问题？怎么解决
|--
在高并发场景下，如果出现大量的TIME-WAIT状态，会占用大量本地端口，导致新请求无法建立连接（报错 Can't assign requested address）
解决：
一方面通过开启内核参数net.ipv4.tcp_tw_reuse = 1 配合时间戳（timestamps）实现安全复用处于 TIME-WAIT 状态的端口；
另一方面可以使用长连接，在应用层将短连接优化为Keep-Alive长连接，复用同一个TCP连接，避免连接的频繁建立和中断

HTTP与HTTPS的区别是什么
HTTPS怎么保证安全的
|--- 
HTTPS 通过 CA 数字证书确保服务器身份真实，并利用非对称加密在客户端与服务器间安全地交换密钥（客户端使用公钥加密，服务器私钥解密，得到对称密钥），
随后切换到高效的对称加密对传输数据进行全量加密，同时配合 MAC 摘要算法（JWT的签名就是一种MAC摘要算法）防止数据被中途篡改 
---|

HTTP与HTTPS的性能差异
|---
HTTPS 的性能损耗主要源于建立连接时比 HTTP 多出的 SSL/TLS 握手过程，这需要额外的网络往返（RTT）来交换证书和密钥，
且非对称加密计算会消耗更多服务器 CPU 资源；
但在连接建立后的数据传输阶段，得益于现代 CPU 的硬件加速和 HTTP/2 多路复用技术的强制应用，HTTPS 的实际运行速度往往能与 HTTP 持平甚至更优
---|

tcp拥塞避免和慢启动是什么
|---
慢启动是指 TCP 连接开始时，先发送少量数据包，每收到一个确认（ACK）就把发送量翻倍，以极快的速度探测网络承载上限；
拥塞避免是当发送速度达到预设阈值（ssthresh）后，将翻倍增长改为线性加1的平缓增长，从而平稳探测极限，防止因发送过猛导致网络拥塞和丢包。
